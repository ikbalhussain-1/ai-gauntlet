# Getting Started

Read this before anything else. It takes 3 minutes.

---

## Step 1 — Extract the kit

Unzip `ai-gauntlet.zip`. You will get a folder called `ai-gauntlet/`.

This folder **is your project**. You will build your application inside it. The prompt files and data files live here alongside your code — that is intentional.

```
ai-gauntlet/           ← your working directory for the entire session
├── prompts/           ← copy-paste prompts for each phase
├── data/              ← sample data for development and scoring
├── scripts/           ← the scoring script
├── README.md          ← full phase-by-phase instructions
├── PROBLEM.md         ← the BRD you will work from
├── PHILOSOPHY.md      ← why this approach eliminates hallucination (read this)
└── RULES.md           ← session rules
```

---

## Step 2 — Open Claude Code

```bash
cd ai-gauntlet
claude --dangerously-skip-permissions
```

The `--dangerously-skip-permissions` flag is YOLO mode. Claude will act without asking you to approve each action. This is Rule 3 of the session — it is required, not optional.

If you are not comfortable with this flag, you can instead press **Shift+Tab** inside Claude Code until the mode shows **Auto-approve**.

---

## Step 3 — Tell Claude where it is

Your first message in Claude Code — every time you open a new session — should be:

```
Read CLAUDE.md and PROBLEM.md. Summarise what we are building and 
what the next step is.
```

This loads the project context into Claude's working memory. All agents you spawn in this session share this context because CLAUDE.md is in the root.

---

## Step 4 — Create your GitHub repo

Before Phase 6, you will need a private GitHub repo. Create it now so it's ready:

```bash
gh repo create review-intelligence --private --source=. --remote=origin
```

Or create it on github.com and add the remote manually. Either way — do it before you start so there are no surprises at the end.

---

## Step 5 — Set your API key

Make sure your Anthropic API key is available:

```bash
export ANTHROPIC_API_KEY=your_key_here
```

Add this to your `.env` file too (already has a placeholder):
```bash
# In .env:
ANTHROPIC_API_KEY=your_key_here
```

Verify it is set:
```bash
echo $ANTHROPIC_API_KEY  # should print your key, not blank
```

---

## Step 6 — Run the prerequisites check

```bash
bash scripts/prereqs.sh
```

Fix anything red. Do not start building until all critical checks pass.

---

## You are ready when:
- [ ] You are inside the `ai-gauntlet/` directory
- [ ] Claude Code is open with `--dangerously-skip-permissions`
- [ ] `ANTHROPIC_API_KEY` is set
- [ ] `bash scripts/prereqs.sh` shows no red items
- [ ] You have a private GitHub repo ready

**Now read `PHILOSOPHY.md`, then open `README.md` and start Phase 0.**

---

## What the folder will look like after you build

Claude will create these directories inside `ai-gauntlet/`:

```
ai-gauntlet/
├── backend/        ← your API server (created in Phase 1)
├── frontend/       ← your React app (created in Phase 1)
├── tests/          ← your test suite (created in Phase 2)
├── infra/          ← docker-compose and nginx (created in Phase 5)
├── docs/           ← wireframe exports from Stitch (created in Phase 3)
├── PRD.md          ← your product requirements (created in Phase 0)
├── qa-checklist.md ← your UI test checklist (created in Phase 3)
└── SCORE_REPORT.md ← generated by check.sh
```

The kit files (`prompts/`, `data/`, `scripts/`) stay alongside your code. They are part of the project.
